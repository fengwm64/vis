# PRD: Floyd-Warshall All-Pairs Shortest Path Visualization

## 算法定义

Floyd-Warshall 算法用于求解带权有向图中所有顶点对之间的最短路径。算法通过逐步尝试以每个顶点作为中间节点来松弛路径，最终得到全局最短路径矩阵。

### 核心公式

对每个中间节点 k，对每对顶点 (i, j)：
```
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

### 算法边界

- 支持有向带权图，边权可为负数
- 不支持负权环（负权环会导致最短路径无意义）
- 输入规模：顶点数 n ≤ 8（可视化场景下保证动画流畅）
- 边权范围：-99 到 99

### 输入数据

默认示例图（4 个顶点的有向带权图）：

```
顶点: 0, 1, 2, 3
边:
  0 -> 1 (权重 3)
  0 -> 2 (权重 8)
  1 -> 2 (权重 2)
  1 -> 3 (权重 5)
  2 -> 3 (权重 1)
  3 -> 0 (权重 2)
```

初始距离矩阵：
```
     0    1    2    3
0  [ 0,   3,   8,  ∞ ]
1  [ ∞,   0,   2,   5 ]
2  [ ∞,   ∞,   0,   1 ]
3  [ 2,   ∞,   ∞,   0 ]
```

## 可视化步骤

### 主要可视化元素

1. **距离矩阵表格**：核心展示，n×n 矩阵，当前被更新的单元格高亮
2. **图的邻接表示**：可选的图形化展示当前图结构
3. **当前中间节点指示**：明确显示当前以哪个顶点 k 作为中间节点
4. **当前松弛操作**：高亮显示正在比较的 (i, j) 对和经过 k 的路径

### 步骤分解

算法步骤按三层循环展开：

```
For k = 0 to n-1:          // 中间节点
  For i = 0 to n-1:        // 起点
    For j = 0 to n-1:      // 终点
      if dist[i][k] + dist[k][j] < dist[i][j]:
        dist[i][j] = dist[i][k] + dist[k][j]  // 松弛
```

每个步骤记录：
- 当前中间节点 k
- 当前检查的 (i, j) 对
- 是否进行了松弛操作
- 更新前后的 dist[i][j] 值
- 当前完整距离矩阵快照

### 交互控件

- **播放/暂停**：自动逐步播放动画
- **单步前进**：手动前进到下一步
- **单步后退**：回退到上一步
- **重置**：回到初始状态
- **速度控制**：调节动画播放速度

### 说明面板

每个步骤需要显示：
- 当前中间节点 k 的编号
- 当前检查的顶点对 (i, j)
- 比较表达式：`dist[i][k] + dist[k][j]` vs `dist[i][j]`
- 操作结果："松弛成功" 或 "无需更新"
- 当前处于第几轮中间节点迭代

## 复杂度说明

- 时间复杂度：O(n³)，三重循环
- 空间复杂度：O(n²)，距离矩阵存储
- 可视化步骤数：n³ 个基本操作（n 个中间节点 × n × n 个顶点对）
