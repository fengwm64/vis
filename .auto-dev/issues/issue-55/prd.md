# PRD: 匈牙利算法 / KM 算法 可视化

## 算法定义

Kuhn-Munkres（KM）算法，也称匈牙利算法的加权版本，用于求解二分图的最大权完美匹配（Assignment Problem）。本可视化实现 KM 算法的完整流程，同时支持无权最大匹配作为退化情况。

## 算法边界

- 输入：带权二分图，左部节点集合 X 和右部节点集合 Y，|X| = |Y| = n（n >= 1），边权 w(x, y) 为非负整数。
- 输出：最大权完美匹配 M，以及总权重 sum。
- 算法保证在 O(n^3) 时间内找到最优解。
- 不处理负权边（可通过偏移处理，但本可视化不展示）。
- 不处理非完全二分图（缺失边视为权 0）。

## 输入规模与示例数据

- 推荐默认规模：n = 4（4x4 二分图），足够展示算法核心过程且不会过于复杂。
- 最大支持规模：n = 6（受可视化布局限制）。
- 最小规模：n = 2。

预置示例数据：
```
示例 1（经典 4x4）：
  X0: [2, 3, 1, 4]
  X1: [3, 2, 4, 1]
  X2: [1, 4, 2, 3]
  X3: [4, 1, 3, 2]

示例 2（简单 3x3）：
  X0: [1, 2, 3]
  X1: [3, 1, 2]
  X2: [2, 3, 1]
```

## 算法步骤（可视化分解）

KM 算法的每一步都应产生一个可暂停的动画帧：

### 1. 初始化阶段
- 展示二分图：左侧 X 节点，右侧 Y 节点，边标注权重。
- 初始化顶标：l(x_i) = max_j w(x_i, y_j)，l(y_j) = 0。
- 高亮显示每个 X 节点的初始顶标值。

### 2. 构建等价子图
- 根据当前顶标构建等价子图：边 (x, y) 属于等价子图当且仅当 l(x) + l(y) = w(x, y)。
- 高亮等价子图的边（与普通边用不同颜色区分）。

### 3. 匈牙利树搜索（找增广路）
- 从当前匹配中未匹配的 X 节点出发，用 BFS/DFS 寻找增广路。
- 展示搜索过程：交替树的扩展、交错边和非交错边。
- 标记 S 集合（已访问的 X 节点）和 T 集合（已访问的 Y 节点）。

### 4. 顶标调整
- 当增广路不存在时，计算 delta = min{l(x) + l(y) - w(x, y) | x in S, y not in T}。
- 更新顶标：对 S 中的 x，l(x) -= delta；对 T 中的 y，l(y) += delta。
- 高亮显示顶标变化和因调整而新增的等价子图边。

### 5. 匹配更新
- 找到增广路后，沿着增广路翻转匹配状态。
- 高亮显示新增的匹配边和被移除的匹配边。
- 更新匹配计数和总权重。

### 6. 终止
- 当匹配数达到 n 时，算法终止。
- 高亮显示最终的完美匹配和总权重。

## 可视化布局与交互控件

### 布局
- 左侧面板：算法信息（当前阶段、顶标值表、匹配列表、总权重）。
- 中央区域：二分图可视化，节点用圆角矩形，边用线条 + 权重标签。
- 右侧面板或底部：步骤说明（当前步骤的文字描述）。

### 交互控件
- **播放/暂停**：自动逐步执行算法。
- **单步前进**：执行一步算法。
- **单步后退**：回退一步（需要保存状态历史）。
- **重置**：回到初始状态。
- **速度控制**：调整自动播放速度。
- **示例切换**：在预置示例之间切换。

### 视觉编码
- 节点颜色：默认灰色，匹配中节点为蓝色，搜索中节点为橙色。
- 边颜色：默认灰色，等价子图边为绿色，匹配边为红色加粗，增广路为橙色。
- 顶标显示：节点旁标注当前顶标值。

## 复杂度说明

- 时间复杂度：O(n^3)，n 为二分图一侧的节点数。
- 空间复杂度：O(n^2)，存储权重矩阵和状态历史。

## 验收清单

1. 算法能正确求解 4x4 和 3x3 的最大权完美匹配。
2. 可视化正确展示每一步的顶标变化、等价子图更新和匹配调整。
3. 播放/暂停、单步前进、单步后退、重置功能正常工作。
4. 自动播放不会出现定时器泄漏（卸载组件时清除定时器）。
5. 步骤说明与当前算法状态一致。
6. 节点和边的视觉编码清晰、颜色区分明显。
7. `npm run build` 通过。
8. 算法模块可通过 Node ESM import 运行自检（`runAlgorithmTests()`）。
9. 卡片内容区域有正常顶部留白（不使用 `pt-0`）。
10. 控件无重复、无死按钮、文案与行为一致。
