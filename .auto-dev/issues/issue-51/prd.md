# PRD: 拓扑排序可视化 (Issue #51)

## 1. 算法定义

拓扑排序（Topological Sort）是在有向无环图（DAG）中，求一个满足所有依赖关系的节点线性序列。对于图中的每条有向边 (u, v)，u 在序列中必须出现在 v 之前。

### 采用算法：Kahn's Algorithm（BFS 方法）

原因：Issue 描述明确要求"不断删除入度为 0 的节点"，这正是 Kahn's 算法的核心思路。该方法步骤直观，每一步都有明确的可视化动作。

**算法流程：**
1. 计算所有节点的入度（in-degree）
2. 将所有入度为 0 的节点加入队列
3. 循环：
   a. 从队列取出一个节点，加入拓扑序列
   b. 对该节点的每个邻居，将其入度减 1
   c. 若邻居入度变为 0，加入队列
4. 若拓扑序列长度等于节点数，排序成功；否则图中存在环

### 边界条件
- 空图：返回空序列
- 单节点无边图：返回该节点
- 存在环的图：算法检测到环，输出错误提示（序列长度 < 节点数）
- 多个入度为 0 的节点：不同取出顺序产生不同合法拓扑序

## 2. 输入规模与示例数据

### 默认示例（课程依赖图）
```
节点: [0, 1, 2, 3, 4, 5]
边:   0→2, 1→2, 2→3, 2→4, 3→5, 4→5
含义: 课程 0 和 1 是课程 2 的先修，课程 2 是 3 和 4 的先修，3 和 4 是 5 的先修
```
预期拓扑序：`[0, 1, 2, 3, 4, 5]`（不唯一）

### 备选示例（含多入度为 0 节点）
```
节点: [A, B, C, D, E]
边:   A→C, B→C, C→D, D→E
```
预期拓扑序：`[A, B, C, D, E]` 或 `[B, A, C, D, E]`

### 含环示例（用于演示检测）
```
节点: [0, 1, 2]
边:   0→1, 1→2, 2→0
```
预期结果：检测到环，排序失败

### 输入限制
- 节点数：2 ~ 10
- 边数：0 ~ 20
- 节点标签：单个大写字母或数字

## 3. 可视化步骤与交互控件

### 可视化区域
- **图展示**：用节点圆圈 + 有向箭头展示 DAG，布局使用力导向或手动预设位置
- **入度表**：实时展示每个节点的当前入度
- **队列**：展示当前待处理的入度为 0 节点队列
- **拓扑序列**：展示已排序的节点序列

### 颜色编码
- **默认节点**：灰色/蓝色
- **入度为 0 的节点（待处理）**：高亮黄色
- **当前处理节点**：橙色
- **已排序节点**：绿色
- **被移除的边**：虚线或渐隐
- **环检测失败**：红色高亮

### 每一步动画
1. **高亮**当前入度为 0 的节点（黄色）
2. **取出**队首节点，变为橙色
3. **移除**该节点出发的所有边（虚线/渐隐），邻居入度减 1
4. **更新**入度表和队列
5. **将**该节点加入拓扑序列，变为绿色
6. 若邻居入度变为 0，高亮为黄色并加入队列

### 交互控件
- **播放/暂停**：自动播放动画
- **单步前进**：执行一步
- **单步后退**：回退一步
- **重置**：回到初始状态
- **速度控制**：调整自动播放速度（快/中/慢）
- **示例切换**：在默认示例、备选示例、含环示例之间切换

## 4. 复杂度说明

- 时间复杂度：O(V + E)，V 为节点数，E 为边数
- 空间复杂度：O(V + E)
- 对于前端可视化，输入规模限制在 10 节点以内，性能无忧
