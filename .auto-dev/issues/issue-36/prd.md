## 算法定义与边界

**插入排序**：将待排列元素划分为「已排序」和「未排序」两部分，每次从未排序部分取出第一个元素，将其插入到已排序部分的正确位置。

- 数组长度：1 ≤ n ≤ 50
- 元素值：-999 ≤ val ≤ 999（整数）
- 时间复杂度：O(n²) 最坏/平均，O(n) 最好（已排序）
- 空间复杂度：O(1)
- 稳定性：稳定

## 输入规模与示例数据

默认示例数组：`[38, 27, 43, 3, 9, 82, 10]`

支持用户自定义数组输入和随机生成。

## 可视化步骤

每一步记录：

1. **当前数组状态**
2. **已排序区间**（高亮显示已排序部分）
3. **当前正在处理的元素**（从未排序部分取出的元素）
4. **比较位置**（正在与已排序部分的哪个元素比较）
5. **插入/移位动作**（元素插入或向右移位）
6. **描述文字**（当前步骤做了什么）

### 步骤生成逻辑

1. 初始状态：整个数组标记为未排序，已排序区间为 `[0, 0)`（空）。
2. 从索引 i=1 开始，将 `arr[i]` 作为 key。
3. 将 key 与已排序部分从右到左逐一比较：
   - 若 `arr[j] > key`，将 `arr[j]` 右移一位（记录移位步骤）。
   - 若 `arr[j] ≤ key` 或已到数组头部，找到插入位置。
4. 将 key 插入到正确位置（记录插入步骤）。
5. 已排序区间扩展到 `[0, i+1)`。
6. 重复直到 i = n-1。

## 交互控件

参照 bubble-sort 模式：

- **播放/暂停**：自动播放所有步骤
- **单步前进**：执行下一步
- **单步后退**：回退到上一步
- **重置**：回到初始状态
- **速度控制**：调节自动播放速度
- **自定义数组输入**：用户输入自定义数组
- **随机生成**：生成随机数组
