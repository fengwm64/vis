# PRD: Issue #30 — PageRank 节点间距过小修复

## 目标动画
- **ID**: pagerank
- **路径**: `/animations/pagerank`
- **现有文件**: `src/animations/pagerank_process_animation.jsx`

## 问题描述

当前 5 个节点的坐标分布过于集中，导致：
1. 节点之间的间距过小，几乎看不到边（连线）
2. 当节点因 PageRank 值较高而变大时（半径最大可达 64px），节点容易重叠
3. 有向边被节点遮挡，影响可视化效果

### 当前节点坐标
```
A: (120, 100)
B: (330, 70)
C: (460, 230)
D: (220, 300)
E: (80, 250)
```

SVG viewBox: `0 0 560 380`

节点主要聚集在左侧（x: 80-460），右侧大量空白未利用。垂直方向也未充分利用（y: 70-300，而 viewBox 高度为 380）。

### 继续修复来源
- **原 Issue**: #26 — 允许用户自定义节点、边和权重
- **原 PR**: #29 — Auto-fix: fix visualization for issue #26
- **上次修复可能遗漏的原因**: PR #29 专注于添加用户自定义功能，未调整基础布局坐标。节点坐标沿用了初始版本的值，随着可视化需求变化，原始布局已不适应当前的展示需求。

## 修复方案

### 需要修改的文件
- `src/animations/pagerank_process_animation.jsx`

### 具体修改内容

1. **扩大节点坐标分布**：将 5 个节点的坐标重新分布，使其在 viewBox `0 0 560 380` 内均匀展开，确保任意两个相邻节点之间至少有 150px 的水平或垂直间距。

2. **保持图的拓扑结构不变**：edges 数组、节点 ID、算法逻辑均不修改。

3. **确保边可见**：调整坐标后，所有 8 条有向边应清晰可见，不被节点遮挡。

### 建议的新坐标（frontend 可微调）
```
A: (140, 80)
B: (380, 80)
C: (460, 260)
D: (280, 340)
E: (100, 260)
```
这样节点在 viewBox 内形成一个更分散的五边形，间距均匀。

## 复现方式
1. 打开 `/animations/pagerank` 页面
2. 观察有向链接图区域
3. 注意节点 A、B、D 之间的间距，以及边是否清晰可见
4. 点击"播放"后观察节点变大时是否重叠
