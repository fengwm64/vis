# PRD: Hopcroft-Karp 二分图最大匹配

## 算法定义

Hopcroft-Karp 算法用于求解二分图的最大基数匹配。相比朴素的匈牙利算法逐条寻找增广路，Hopcroft-Karp 通过 BFS 分层一次性找到所有最短增广路，再用 DFS 批量增广，将时间复杂度从 O(VE) 降低到 O(E√V)。

### 核心概念

- **二分图**：顶点分为左集 U 和右集 V，边只连接 U 和 V 之间的顶点。
- **匹配**：边集 M，其中任意两条边不共享端点。
- **增广路**：从左集未匹配点出发、交替经过非匹配边和匹配边、终止于右集未匹配点的路径。
- **BFS 分层**：从所有左集未匹配点出发做 BFS，按层标记距离，只沿可行边前进，找到到达右集未匹配点的最短距离。
- **DFS 增广**：在 BFS 构建的分层图上，从左集未匹配点出发做 DFS，沿距离递增的路径寻找增广路，找到后翻转匹配。

### 算法边界

- 输入：二分图，由左集节点列表、右集节点列表和边列表定义。
- 节点标签：字符串（如 "U1", "U2", "V1", "V2"）。
- 边：无向，每条边连接一个左集节点和一个右集节点。
- 输出：最大匹配的边集。
- 不处理带权匹配（非 KM / 加权匹配问题）。

### 输入规模与示例

默认示例：
- 左集 U: ["U1", "U2", "U3", "U4"]
- 右集 V: ["V1", "V2", "V3", "V4"]
- 边: [("U1","V1"), ("U1","V2"), ("U2","V1"), ("U2","V3"), ("U3","V2"), ("U3","V4"), ("U4","V3")]

预期最大匹配: 4（完美匹配）

用户可自定义输入（通过文本框编辑左集、右集和边列表），输入规模建议：左右集各 ≤ 10 个节点，边数 ≤ 30。

## 可视化步骤

### 动画主循环

每轮迭代包含两个阶段：

**阶段 1：BFS 分层**
1. 初始化：标记所有左集未匹配点，距离设为 0，其余为 ∞。
2. 逐层 BFS：从当前层出发，沿可行边（非匹配边 → 匹配边交替）扩展到下一层。
3. 可行边定义：从左集节点出发走非匹配边到右集，从右集节点走匹配边回左集。
4. BFS 终止条件：到达右集未匹配点，记录最短增广路长度 dist。
5. 如果没有增广路，算法终止。

**阶段 2：DFS 批量增广**
1. 从每个左集未匹配点出发做 DFS。
2. 沿距离递增 1 的路径前进（严格遵循 BFS 分层）。
3. 找到增广路后翻转匹配边（非匹配变匹配，匹配变非匹配）。
4. 标记已使用的节点，避免同轮重复使用。

### 单步数据结构

每步应包含：
- `phase`: "bfs" 或 "dfs"
- `step`: 当前步骤编号
- `bfsLayer`: BFS 分层信息（节点 → 距离值），仅 BFS 阶段有
- `currentNode`: 当前正在处理的节点
- `exploringEdge`: 当前正在探索的边 [from, to]
- `matching`: 当前匹配边集
- `dist`: 距离数组（左集和右集）
- `augmentingPath`: 当前找到的增广路径（DFS 阶段）
- `foundAugmenting`: 是否找到增广路
- `description`: 当前步骤的文字说明

### 交互控件

- **播放/暂停**：自动推进步骤。
- **单步前进**：推进一步。
- **单步后退**：回退一步。
- **重置**：回到初始状态。
- **速度控制**：滑动条调节自动播放速度。
- **输入编辑**：允许用户修改左集、右集和边列表，点击"重新计算"生成新步骤。

## 复杂度说明

- 时间复杂度：O(E√V)，每轮 BFS+DFS 至少找到一条最短增广路，最多 O(√V) 轮。
- 空间复杂度：O(V+E)，存储图、距离数组和匹配信息。

1. 算法能正确计算默认示例的最大匹配数（4）。
2. 算法能处理不完美匹配的情况（如删除某些边后匹配数 < |U|）。
3. 算法能处理空图（无边）、单边、完全二分图等边界情况。
4. BFS 分层过程有清晰的可视化：节点按层高亮，探索边有动画。
5. DFS 增广过程有清晰的可视化：增广路径高亮，匹配翻转有动画。
6. 匹配边和非匹配边有明显视觉区分（颜色或线型）。
7. 播放/暂停/单步前进/单步后退/重置功能正常。
8. 自动播放不会出现定时器泄漏。
