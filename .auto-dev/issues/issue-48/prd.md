## 算法定义

Bellman-Ford 算法用于求解带权有向图中的单源最短路径问题。与 Dijkstra 不同，Bellman-Ford 可以处理包含负权边的图，并能检测负权环。

**核心思想**：对图中所有边进行 V-1 轮松弛操作（V 为顶点数）。每一轮遍历所有边，尝试通过当前边缩短路径距离。若第 V 轮仍有距离更新，说明存在负权环。

**时间复杂度**：O(V * E)，其中 V 为顶点数，E 为边数。

## 算法边界与约束

- 图为有向加权图，邻接表表示，权重可为负数
- 无自环（简化实现）
- 源节点可达性：不可达节点距离保持 Infinity
- 负权环检测：若第 V 轮仍有松弛更新，报告存在负权环
- 输入规模：建议默认示例 6 节点、8-10 条边，便于可视化

## 输入规模与示例数据

默认示例图（6 节点、9 条边，含负权边，无负权环）：

```
A → B: 6
A → C: 4
A → D: 5
B → E: -1
C → B: -2
C → E: 3
D → C: -2
D → F: -1
E → F: 3
```

源节点：A

预期最短距离：
- A: 0, B: 1, C: 3, D: 5, E: 0, F: 3

## 可视化步骤

### 算法步骤数据结构

每个步骤应包含：
- `step`: 步骤编号
- `iteration`: 当前松弛轮次（0 = 初始化，1 ~ V-1 = 松弛轮，V = 负权环检测轮）
- `distances`: 当前各节点距离表
- `prev`: 前驱节点表（用于路径回溯）
- `relaxedEdge`: 当前正在松弛的边 [u, v]，null 表示无
- `updatedNode`: 距离被更新的节点，null 表示无
- `description`: 本步骤的文字说明
- `isNegativeCycleCheck`: 是否为负权环检测轮

### 步骤生成流程

1. **初始化**：源节点距离 0，其余 ∞，前驱 null
2. **第 1 轮松弛**：遍历所有边，松弛 A→B(6), A→C(4), A→D(5), B→E(-1+6=5), C→B(4-2=2), C→E(4+3=7), D→C(5-2=3), D→F(5-1=4), E→F(5+3=8)
3. **第 2 轮松弛**：继续更新，C→B 使 B 距离更新为 3-2=1
4. **第 3~5 轮松弛**：继续收敛
5. **负权环检测**：再遍历一次所有边，确认无更新
