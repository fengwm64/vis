## 1. 算法定义与边界

### 算法描述
快速排序（Quick Sort）是一种分治排序算法，通过选择一个 pivot 元素将数组分为两部分：小于 pivot 的元素和大于 pivot 的元素，然后递归排序两部分。

### 核心步骤
1. **选择 Pivot**：从当前子数组中选择一个元素作为基准（本实现选择第一个元素）
2. **分区（Partition）**：重新排列数组，使得所有小于 pivot 的元素在其左侧，所有大于 pivot 的元素在其右侧
3. **递归**：对左右两个子数组重复上述过程

### 算法边界
- 输入：整数数组（可包含负数、重复值）
- 输出：升序排列的数组
- 空数组或单元素数组视为已排序
- 最坏情况：O(n²)（已排序数组，pivot 选择不当）
- 平均情况：O(n log n)
- 空间复杂度：O(log n)（递归栈）

## 3. 可视化步骤与交互控件

### 可视化步骤数据结构

每个步骤对象应包含：

```javascript
{
  array: number[],           // 当前数组状态
  pivot: number | null,      // 当前 pivot 的索引（null 表示无 pivot）
  range: [number, number],   // 当前处理的子数组范围 [lo, hi]
  comparing: number | null,  // 正在比较的元素索引
  swapping: [number, number] | null, // 正在交换的两个索引
  partitioned: number[],     // 已分区完成的索引（最终位置确定）
  sorted: number[],          // 已排序完成的索引
  description: string,       // 当前步骤的文字说明
  comparisons: number,       // 累计比较次数
  swaps: number,             // 累计交换次数
  recursionDepth: number     // 当前递归深度
}
```

### 关键可视化节点

1. **初始状态**：展示原始数组，无 pivot
2. **选择 Pivot**：高亮显示选中的 pivot 元素
3. **分区过程**：
   - 比较当前元素与 pivot（高亮 comparing）
   - 交换元素位置（高亮 swapping）
   - 标记已分区区域
4. **分区完成**：pivot 到达最终位置，标记为 sorted
5. **递归调用**：高亮当前处理的子数组范围
6. **排序完成**：所有元素标记为 sorted

### 交互控件

- **播放/暂停**：自动播放动画
- **单步前进**：前进一个步骤
- **单步后退**：后退一个步骤
- **重置**：回到初始状态
- **速度控制**：调整动画播放速度
- **随机生成**：生成新的随机数组
- **自定义输入**：手动输入数组

## 4. 复杂度说明

### 时间复杂度
- **最好情况**：O(n log n) — 每次分区均匀
- **平均情况**：O(n log n)
- **最坏情况**：O(n²) — 每次分区极不平衡（已排序数组）

### 空间复杂度
- **平均**：O(log n) — 递归栈深度
- **最坏**：O(n) — 退化为链表

### 可视化步骤数
- 平均：~2n log n 步（比较 + 交换 + 状态更新）
- 最坏：~n² 步
