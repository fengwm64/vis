# PRD: [auto-fix] PageRank 过程动画 — 增加有向图的箭头可见性

## 目标动画
- **ID**: pagerank
- **标题**: PageRank 过程动画
- **路径**: /animations/pagerank
- **源文件**: `src/animations/pagerank_process_animation.jsx`

## 问题描述

PageRank 动画的有向链接图在视觉上难以辨认方向。虽然代码中已有 `Arrow` 组件绘制了线条和三角形箭头，但箭头存在以下视觉问题：

1. **箭头过小**：polygon 仅 10px 宽、10px 高（`"0,-5 10,0 0,5"`），在 SVG viewBox 560×380 的画布上几乎不可见。
2. **非活跃状态对比度低**：未播放且 round=0 时，线条颜色 `#a8a8a8`，opacity 0.55，箭头与线条同色同透明度，辨识度极低。
3. **箭头与线条无区分**：箭头和线条使用完全相同的颜色和透明度，箭头形状不突出。

## 修改范围

仅修改 `src/animations/pagerank_process_animation.jsx` 中的 `Arrow` 组件，不涉及算法逻辑。

### 需要修改的内容

**`Arrow` 组件（第 90-135 行）**：

1. **增大箭头尺寸**：将 polygon points 从 `"0,-5 10,0 0,5"` 改为更大的尺寸（如 `"0,-7 14,0 0,7"` 或 `"0,-8 16,0 0,8"`），使箭头在画布上清晰可辨。
2. **提高非活跃状态可见性**：
   - 线条颜色从 `#a8a8a8` 改为更深的颜色（如 `#94a3b8` slate-400 或 `#64748b` slate-500）。
   - 线条 opacity 从 0.55 提高到 0.7 或更高。
   - 箭头 fill 使用比线条更深的颜色或更大的对比度。
3. **确保箭头始终可辨认**：即使在 round=0、未播放状态下，用户也能清楚看到链接的方向。

### 不需要修改的内容

- 算法逻辑（`computeIterations`、`runPageRankTests`）
- 节点渲染
- 排名面板
- 控制按钮
- 布局结构
- meta.js 或其他文件
