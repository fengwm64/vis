# PRD: Kruskal 最小生成树可视化

## 算法定义

Kruskal 算法是一种贪心算法，用于在带权无向连通图中求最小生成树（MST）。核心思想：

1. 将所有边按权重从小到大排序。
2. 依次遍历排序后的边，若该边连接的两个顶点属于不同连通分量（不成环），则将该边加入 MST，并合并两个连通分量。
3. 重复直到 MST 包含 `V-1` 条边。

关键数据结构：**并查集（Union-Find）**，支持路径压缩和按秩合并。

## 算法边界与约束

- 图为带权无向连通图，顶点数 V >= 2，边数 E >= V-1。
- 边权为正整数。
- 图中不存在重边（同一对顶点间最多一条边）。
- 输入规模：V ∈ [4, 8]，E ∈ [5, 15]，适合可视化展示。
- 算法保证能找到 MST（图连通）。

## 输入规模与示例数据

默认示例图（6 个顶点，10 条边）：

```
顶点: A, B, C, D, E, F
边:
  A-B: 4, A-C: 2, B-C: 1, B-D: 5
  C-D: 8, C-E: 10, D-E: 3, D-F: 6
  E-F: 7, B-F: 9
```

排序后边序列：B-C(1), A-C(2), D-E(3), A-B(4), B-D(5), D-F(6), E-F(7), C-D(8), B-F(9), C-E(10)

MST 结果：B-C(1), A-C(2), D-E(3), A-B(4), D-F(6)，总权重 16。

## 可视化步骤

每一步对应"考虑一条边"的决策：

| 步骤 | 动作 | 可视化展示 |
|------|------|-----------|
| 初始 | 展示原图，所有边为灰色 | 邻接信息面板 |
| 排序 | 显示排序后的边列表 | 边列表高亮排序结果 |
| 考虑边 | 当前考虑的边高亮为黄色 | 说明面板显示"考虑边 X-Y，权重 W" |
| 接受边 | 边变为绿色，加入 MST | 并查集状态更新，MST 边数+1 |
| 拒绝边 | 边变为红色虚线（成环） | 说明面板显示"X 和 Y 已在同一连通分量，跳过" |
| 完成 | MST 边全部绿色，显示总权重 | 汇总面板 |

## 交互控件

- **播放/暂停**：自动逐步执行。
- **单步前进**：推进一步（考虑下一条边）。
- **单步回退**：回退一步。
- **重置**：回到初始状态。
- **速度控制**：调节自动播放速度。

不添加"随机生成图"按钮——保持与现有动画一致的简洁控件风格。

## 复杂度说明

- 时间复杂度：O(E log E)，瓶颈在边排序。
- 空间复杂度：O(V + E)，并查集和边存储。
- 可视化步骤数：O(E)，每条边恰好考虑一次。
