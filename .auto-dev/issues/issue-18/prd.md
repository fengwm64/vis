## 1. 算法定义

Dijkstra 算法用于求解带权有向图中，从单源节点到所有其他节点的最短路径。要求所有边权为非负数。

**核心思想**：维护一个优先队列（小顶堆），每次取出当前距离最小的未确定节点，对其邻居执行松弛操作（relaxation），直到所有可达节点都被确定。

**算法步骤**：
1. 初始化：源节点距离为 0，其余节点距离为 ∞。
2. 将所有节点加入待处理集合（未确定集合）。
3. 循环：从待处理集合中选出距离最小的节点 u，标记为"已确定"。
4. 对 u 的每个邻居 v，计算 `dist[u] + weight(u,v)`，若小于当前 `dist[v]`，则更新 `dist[v]` 和 `prev[v]`。
5. 重复步骤 3-4，直到待处理集合为空。

## 2. 边界与约束

- 图的节点数：5-8 个（可视化空间限制，保证清晰可读）。
- 边权：正整数，范围 1-20。
- 图结构：有向图，允许单向边和双向边。
- 不存在负权边（Dijkstra 算法前提）。
- 图保证连通（从源节点可达所有节点），避免出现不可达节点导致可视化困惑。
- 源节点固定为用户选择的起始节点（默认第一个节点）。

## 3. 输入规模与示例数据

使用固定示例图，包含 6 个节点（A-F）和约 9-10 条带权有向边：

```
节点: A(源), B, C, D, E, F
边示例:
  A→B (4), A→C (2)
  B→C (1), B→D (5)
  C→D (8), C→E (10)
  D→F (2)
  E→F (3)
  B→F (7)
```

源节点为 A。最短路径结果示例：
- A→A: 0
- A→C: 2 (直接)
- A→B: 4 (直接)
- A→D: 9 (A→B→D)
- A→F: 9 (A→B→D→F)
- A→E: 12 (A→C→E)

## 4. 可视化步骤

每一步的动画状态需包含：
- **当前处理节点**：高亮显示正在被"确定"的节点。
- **已确定节点集合**：用不同颜色标记已完成最短路径确定的节点。
- **距离表**：实时显示从源节点到每个节点的当前最短距离估计。
- **松弛操作**：当发现更短路径时，高亮对应的边和更新的距离值。
- **最短路径树**：逐步构建，用特殊样式标记已确定的最短路径边。

**步骤数据结构**（算法模块输出）：
```js
{
  step: 0,           // 步骤序号
  currentNode: "A",  // 当前处理节点
  distances: { A: 0, B: 4, C: 2, D: Infinity, E: Infinity, F: Infinity },
  visited: [],       // 已确定节点
  relaxedEdge: null, // 本步被松弛的边 [from, to]
  updatedNode: null, // 距离被更新的节点
  prev: { A: null, B: "A", C: "A", D: null, E: null, F: null }, // 前驱节点（最短路径树）
  description: "初始化：源节点 A 距离为 0"
}
```

## 5. 交互控件

参照 PageRank 动画模式：

- **单步前进**：手动推进一步。
- **重置**：回到初始状态。
- **速度控制**：不需要额外滑块，保持简洁。

不需要"单步后退"按钮（PageRank 也没有），保持控件一致性。

## 6. 复杂度说明

- 时间复杂度：O((V + E) log V)，使用优先队列。
- 空间复杂度：O(V)。
- 对于 6 节点的小图，计算量极小，步骤数约 6-12 步。
