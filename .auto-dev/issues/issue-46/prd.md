# PRD: Issue #46 - 插入排序图例冗余修复

## 目标动画

- ID: `insertion-sort`
- 路径: `/animations/insertion-sort`
- 现有文件:
  - `src/animations/insertion-sort/index.jsx` (需修改)
  - `src/animations/insertion-sort/algorithm.js` (无需修改)
  - `src/animations/insertion-sort/meta.js` (无需修改)

## 问题描述

图例区域存在两条含义高度重叠的条目：

| 条目 | 样式 | 含义 |
|------|------|------|
| `当前取出` | `bg-rose-400` 实色方块 | 当前被取出的 key 元素所在柱子的颜色 |
| `key 已取出` | `border-2 border-dashed border-slate-300 bg-slate-100` 虚线方块 | key 被取出后留下的空位占位符 |

两条都表达"key 已被取出"这一概念，用户容易混淆，属于冗余。

## 继续修复来源

- 原 Issue: #40
- 原 PR: 未提供
- 上次修复可能遗漏原因假设: 上次修复可能聚焦于算法步骤或动画逻辑，未审查图例区域的文案一致性。

## 修复方案

合并或重新措辞图例条目，消除语义重复：

1. **保留 `当前取出`** 条目（rose-400），描述 key 元素本身。
2. **重新措辞 `key 已取出`** 条目，改为明确描述其视觉含义：`空位占位` 或 `key 原位`，表示 key 被取出后数组中留下的空槽位。
3. 确保两条图例的含义不再重叠，各自对应一种独立的视觉状态。

## 验收清单

1. 图例区域不再出现"key 已取出"和"当前取出"含义重复的条目。
2. 修改后的图例文案准确描述各自对应的视觉样式。
3. `npm run build` 通过。
4. 动画功能不受影响：播放、暂停、单步前进/后退、重置、自动播放均正常。
5. 空位占位符（虚线边框 ∅）在动画过程中正确显示。
