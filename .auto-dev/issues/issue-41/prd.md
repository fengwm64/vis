# PRD: BFS 遍历算法可视化

## 算法定义

BFS（Breadth-First Search，广度优先搜索）是一种图遍历算法，从指定起始节点出发，逐层访问图中的所有节点。使用队列（FIFO）管理待访问节点，确保按层级顺序遍历。

## 算法边界

- 输入：无向图（节点集合 + 边集合）+ 起始节点
- 输出：遍历顺序序列
- 图连通性：不要求全连通，遍历从起始节点可达的所有节点
- 节点标识：使用大写字母（A、B、C...），最多 8 个节点
- 边：无向边，无权值

## 输入规模和示例数据

默认图结构（6 个节点，7 条边）：

```
节点：A, B, C, D, E, F
边：A-B, A-C, B-D, B-E, C-E, C-F, E-F
```

起始节点：A

预期 BFS 遍历顺序（A 出发）：A → B → C → D → E → F

## 可视化步骤

每一步对应算法的一个原子操作，数据结构应驱动动画状态：

1. **初始状态**：显示完整图结构，所有节点为默认颜色，队列为空
2. **入队起始节点**：起始节点入队，标记为"已入队"（浅色高亮），队列显示 [A]
3. **出队并访问**：从队列头部取出节点，标记为"当前访问"（深色高亮），加入遍历结果序列
4. **探索邻居**：依次高亮当前节点的所有未访问邻居边和节点
5. **入队邻居**：将未访问的邻居节点入队，标记为"已入队"，队列更新
6. **重复 3-5**：直到队列为空
7. **完成**：所有可达节点已访问，显示完整遍历序列

## 交互控件

- **播放/暂停**：自动逐步执行，间隔 1.2 秒
- **单步前进**：执行下一步
- **重置**：回到初始状态，清空队列和遍历结果

## 可视化布局

参考 PageRank 动画的双栏布局：

- 左栏：图结构 SVG，节点按力导向布局，边为无向线段
  - 节点颜色状态：默认（白底）→ 已入队（浅蓝）→ 当前访问（深蓝/靛蓝）→ 已完成（灰色）
  - 当前正在探索的边高亮
- 右栏：
  - 上方卡片：队列状态（显示队列内容，当前出队元素高亮）
  - 下方卡片：遍历结果序列（按访问顺序显示）
  - 底部卡片：算法说明

## 复杂度说明

- 时间复杂度：O(V + E)，V 为节点数，E 为边数
- 空间复杂度：O(V)，队列最多存储所有节点
