# PRD: 二分查找可视化

## Issue
#8 - 二分查找

## 1. 算法定义和边界

### 算法描述
二分查找（Binary Search）在**已排序**数组中查找目标值。每步取中间元素与目标比较，根据比较结果缩小一半搜索范围，直到找到目标或范围为空。

### 算法边界
- 输入：已升序排列的整数数组 + 目标值
- 输出：目标值的索引（找到）或 -1（未找到）
- 前提条件：数组必须已排序（前端保证）
- 元素值范围：-999 ~ 999（支持负数）
- 目标值范围：-999 ~ 999
- 允许数组中无目标值（查找失败情况）
- 允许数组中有重复元素（找到任意一个即可）

### 不变量
- 每步后搜索范围 `[left, right]` 严格缩小
- 目标值若存在，必在 `[left, right]` 范围内
- 终止条件：`left > right`（未找到）或 `arr[mid] == target`（找到）

## 2. 输入规模和示例数据

### 输入规模
- 数组长度：1 ~ 30（过长影响可视化可读性）
- 默认长度：8 个元素
- 元素值范围：-999 到 999

### 示例数据
```
默认数组: [-10, -3, 0, 5, 8, 12, 25, 37]
默认目标: 5
随机生成: 用户指定长度（1~30），生成含正负数的随机数组并排序，目标随机选自数组元素
自定义输入: 用户手动输入逗号分隔的整数，数组自动排序
```

## 3. 可视化步骤和交互控件

### 步骤数据结构
```javascript
{
  step: number,           // 步骤序号，0 为初始状态
  array: number[],        // 当前数组（始终不变，已排序）
  left: number,           // 左边界索引
  right: number,          // 右边界索引
  mid: number | null,     // 中间索引（初始状态为 null）
  midValue: number | null,// arr[mid] 的值
  target: number,         // 目标值
  comparison: '<' | '>' | '=' | null,  // 比较结果
  found: boolean,         // 是否找到
  done: boolean,          // 是否结束
  foundIndex: number | null, // 找到时的索引
  description: string     // 本步说明文字
}
```

### 步骤示例
数组 `[-10,-3,0,5,8,12,25,37]`，目标 `12`：
- Step 0: 初始状态，left=0, right=7
- Step 1: mid=3, arr[3]=5 < 12, left=mid+1=4
- Step 2: mid=5, arr[5]=12 == 12, 找到！索引 5

查找失败示例，目标 `99`：
- Step 0: left=0, right=7
- Step 1: mid=3, arr[3]=5 < 99, left=4
- Step 2: mid=5, arr[5]=12 < 99, left=6
- Step 3: mid=6, arr[6]=25 < 99, left=7
- Step 4: mid=7, arr[7]=37 < 99, left=8, left > right, 未找到

### 可视化布局
- **数组区域**：水平排列的元素格，每格显示值和索引
- **指针标记**：left（绿色下标）、right（红色下标）、mid（蓝色高亮）用 Framer Motion 动画移动
- **搜索范围**：left~right 区间的元素保持彩色，范围外的元素变灰
- **说明面板**：每步显示当前 left/right/mid 值、比较结果和下一步操作
- **结果提示**：查找成功时显示成功标记和索引；查找失败时显示未找到提示

### 交互控件
1. **输入控制**
   - 文本输入框：手动输入数组（逗号分隔，自动排序）
   - 目标值输入框
   - 长度选择：设置随机数组长度（1~30）
   - 随机生成按钮：生成含正负数的随机数组，目标随机选自数组
   - 开始查找按钮：触发算法计算步骤

2. **播放控制**（复用 PageRank 模式）
   - 播放/暂停按钮
   - 单步前进按钮
   - 回退按钮
   - 重置按钮
   - 速度控制滑块

3. **说明面板**
   - 当前步骤描述
   - 当前 left/right/mid 值
   - 比较结果
   - 查找结果（成功/失败）

## 4. 复杂度说明

### 时间复杂度
- 最好情况：O(1)，中间元素就是目标
- 最坏情况：O(log n)，每次范围减半
- 平均情况：O(log n)

### 空间复杂度
- O(1)，只用 left/right/mid 三个变量

### 可视化复杂度
- 步骤数量：最多 floor(log2(n)) + 1 步
- 30 个元素最多约 5 步，动画总时长可控
