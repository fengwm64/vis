# PRD: 冒泡排序 — 负数可视化修复

Issue: #9
类型: auto-fix
目标动画: bubble-sort (`/animations/bubble-sort`)

## 问题描述

用户报告负数在冒泡排序可视化中被当作正数处理，例如 -90 的柱高和 90 一样。

经验证，**算法逻辑本身是正确的** — `computeSteps([90, -90, 5, 3])` 正确输出 `[-90, 3, 5, 90]`。问题出在前端可视化组件。

## 根因分析

`src/animations/bubble-sort.jsx` 中：

1. **第 53 行**: `Math.max(...current.array.map(Math.abs), 1)` — 用绝对值计算最大值
2. **第 188 行**: `(Math.abs(value) / maxVal) * 100` — 用绝对值计算柱高

导致 -90 和 90 的柱子高度相同，视觉上无法区分正负数。

## 目标文件

- `src/animations/bubble-sort.jsx` — 唯一需要修改的文件
- `src/animations/bubble-sort/algorithm.js` — 算法无需修改

## 修复方案

在柱状图中支持负数的正确可视化：

1. **计算值域**: 找出数组中的最大值和最小值，计算从 min 到 max 的范围
2. **零线基线**: 当数组包含负数时，在柱状图中显示零线（baseline）
3. **柱高方向**:
   - 正数柱子从零线向上延伸
   - 负数柱子从零线向下延伸
4. **数值标签**: 保持显示原始值（含负号），不做任何转换

具体实现要点：

- `maxVal` 应改为同时考虑正数最大值和负数最小值的绝对值，或改用 range-based 方式
- 柱状图容器需要支持从中间（零线）向上下两个方向绘制
- 柱子颜色逻辑保持不变（比较中/交换中/已就位/未排序）

## 输入规模和示例数据

- 用户可手动输入任意整数数组（含负数）
- 随机生成范围已包含负数: `Math.floor(Math.random() * 198) - 99`（范围 -99 到 98）
- 测试用例: `[-90, 5, 3, -10, 0, 90]`, `[3, -1, 0, -5, 2]`

## 验收清单

1. 输入 `[90, -90, 5, 3]`，-90 的柱子应明显低于 90 的柱子（向下延伸或更短）
2. 输入含负数的数组，排序结果正确（算法不变，仅验证可视化不影响正确性）
3. 零值柱子应有合理的视觉表现
4. 随机生成的数组（已含负数）能正常可视化
5. 不含负数的数组（如默认 `[64, 34, 25, 12, 22, 11, 90, 1]`）可视化效果不变
6. 播放/暂停/单步/回退/重置功能正常
7. 步骤描述中的数值显示正确（含负号）
8. `npm run build` 通过
