# PRD: [auto-fix] 二分查找 - 查找按钮无效

## Issue
#12 - [auto-fix] 二分查找: 查找 按钮无效

## 1. 问题描述

二分查找动画页面的"查找"按钮（位于目标值输入框右侧）存在交互问题：

1. **按钮无效感**：点击"查找"后仅重新计算步骤并重置到第 0 步，但不自动播放动画，用户看不到任何搜索过程，感觉按钮没有效果。
2. **与播放按钮功能重复**：当前"查找"按钮的功能（确认输入 + 重置）与手动修改输入后点击"播放"的体验无明显区别，用户困惑两个按钮的分工。

## 2. 目标动画

- **ID**: binary-search
- **路径**: `/animations/binary-search`
- **现有文件**:
  - `src/animations/binary-search.jsx` (主组件，第 341-346 行为"查找"按钮)
  - `src/animations/binary-search/index.jsx` (re-export)
  - `src/animations/binary-search/meta.js`
  - `src/animations/binary-search/algorithm.js`

## 3. 问题类型

interaction（交互问题），不涉及算法逻辑。

## 4. 修复方案

将"查找"按钮改为**确认输入并自动开始播放**：

1. 点击"查找"后，先执行 `handleConfirm` 解析输入并计算步骤。
2. 确认完成后自动设置 `playing = true`，让动画立即开始播放。
3. 这样"查找"按钮的语义变为"用这些参数开始查找"，与"播放/暂停"按钮（控制已启动动画的暂停/继续）形成清晰分工。

### 预期交互流程

| 操作 | 预期行为 |
|------|----------|
| 修改数组/目标值后点击"查找" | 确认输入 → 计算步骤 → 自动开始播放动画 |
| 点击"播放" | 从当前步骤开始/继续自动播放 |
| 点击"暂停" | 暂停自动播放 |
| 点击"上一步/下一步" | 手动单步控制 |
| 点击"重置" | 回到第 0 步，停止播放 |
| 点击"随机生成" | 生成新数组和目标值（不自动播放） |

## 5. 验收清单

- [ ] 点击"查找"后，动画自动开始播放（无需再点"播放"）
- [ ] "查找"按钮文案保持"查找"，无需修改
- [ ] "播放/暂停"按钮在查找播放过程中可正常暂停
- [ ] 搜索完成后（`current.done`），播放自动停止
- [ ] 修改输入后不点"查找"，点"播放"仍可正常工作（使用旧步骤）
- [ ] "随机生成"按钮不触发自动播放
- [ ] 手动单步操作（上一步/下一步）在查找播放期间可正常使用
- [ ] 无死按钮、无重复按钮、无文案行为不一致
- [ ] `npm run build` 通过
